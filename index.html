Overview

What this project is

It is a complete DIY roaster project, not just a firmware sketch. The repository includes the enclosure, PCB files, popper modification notes, firmware, web interfaces, and operating documentation.

Use accessible parts

The build is based on off-the-shelf components including an ESP32 Nano, Nextion display, SSR, power supplies, and a hot-air popcorn popper.

Control the roast

Temperature and fan targets are driven by interpolated setpoints instead of only manual adjustment during the roast.

Record and refine

Stream roast data to Artisan, tune profiles over time, and use the built-in software tools to repeat successful batches.

Build process

Main steps

01

Print and source parts

Print the enclosure, order the PCB, and gather the display, controller, sensors, popper, and power hardware.

02

Modify the popper

Rework the heater and fan wiring, remove the stock thermostat behavior, and install the roasting chamber thermocouple.

03

Assemble the controller

Mount the PCB, display, power hardware, and wiring harnesses into the case and connect the popper hardware.

04

Flash and configure

Install firmware, upload the Nextion UI, configure WiFi if needed, and begin creating or editing roast profiles.

Software

Control and monitoring features

Profile editor

Edit roast profiles in the browser with time, temperature, and fan setpoints stored on the device.

Console and OTA

Use the web console for live information and update firmware over the network without opening the enclosure.

Artisan integration

Send roast data wirelessly to Artisan so the machine fits into a standard roast logging workflow.

Artisan support Roast telemetry can be recorded and reviewed with the existing Artisan tools included in the repo.

Safety

Built for a high-heat project

The roaster handles line voltage and high temperatures, so the firmware includes explicit safety checks and controlled failure modes.

Temperature limits

Absolute and roast-specific cutoffs stop heating when temperatures exceed defined bounds.

Sensor checks

Thermocouple faults, implausible jumps, and fan-side sensing problems are handled as error conditions.

State-machine handling

Cooling, error, and watchdog behavior are designed to move the machine toward a safer state when something goes wrong.

Build validation

The repository includes targeted compile checks and test sketches for profiles, PID, the state machine, and safety logic.

Get started

Use the repo as a full build guide or as a reference for your own controller.

The repository can be followed directly, or adapted if you already have your own hardware and only want the control software or UI ideas.
