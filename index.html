Why build your own?

Because great roasting at home should feel repeatable, not mysterious.

Fresh coffee on your schedule

Roast only what you need, tune the development level to your taste, and stop settling for stale bags on a shelf.

Data beats guesswork

Profiles, telemetry, and Artisan integration let you learn from every roast instead of starting from zero every weekend.

Affordable hardware, premium behavior

Build path

A clear route from printed parts to first roast.

01

Print and source the hardware

Start with the enclosure, PCB, ESP32 Nano, Nextion display, popcorn popper, SSR, power supplies, connectors, and thermocouple hardware.

02

Modify the popper and wire the control stack

Remove the stock thermostat path, add the thermocouple, route the heater and fan wiring, and mount everything into the printed case.

03

Flash the firmware and configure the UI

Install dependencies, upload the Nextion firmware, flash the ESP32, and bring up the WiFi-enabled control surface.

04

Roast, observe, refine

Use profiles, live telemetry, and Artisan recordings to iterate toward repeatable light, medium, or espresso-style roasts.

Software experience

Modern control surfaces for an approachable DIY machine.

On-device web tools

Use the web console for live debug logs, update firmware over the air, and edit roast profiles in the browser.

Wireless roast logging

Stream roast data to Artisan so your roast curves and notes live in the same workflow you already know.

Profile-driven roasting

Drive temperature and fan targets from user-defined setpoints with interpolation between points, not fixed manual steps.

Artisan-ready WebSocket data streaming makes the roaster fit neatly into an existing roasting workflow.

Safety matters

This is a project for high heat and line voltage, so the software is built to fail safer.

Thermal limits

Absolute and roast-specific over-temperature cutoffs push the machine into safe handling when readings exceed acceptable bounds.

Sensor validation

Thermocouple faults, implausible jumps, and fan-side sensing problems are treated as first-class failure modes.

Cooling and watchdogs

Cooling-state handling, emergency behavior, and a hardware watchdog reduce the chance of a silent bad state.

Tested workflow

There is an Arduino CI build, targeted test sketches, and explicit validation paths for the most safety-sensitive logic.
